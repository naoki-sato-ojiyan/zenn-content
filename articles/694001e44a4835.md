今回は次のオリジナルアプリ「まなびクエスト（ドラクエ風学習ゲーム）」を作るためにJavaScript基礎・Canvas API・Phaser.jsを学びました。

## まなびクエストとは？

```
ドラクエ風RPGゲーム × 学習アプリ

フィールドを歩いて敵に遭遇（ランダムエンカウント）
　　↓
算数・漢字などの問題が出題される
　　↓
正解 → 敵にダメージ！
```

# Part1. JavaScript基礎

## 1. 変数・定数（let・const）

```javascript
// const → 変更不可（定数）
const playerName = 'ゆうしゃ';

// let → 変更可能（変数）
let playerHp = 30;
let playerLevel = 1;
```

### constとletの使い分け

```
const → 名前など変わらないものに使う
let   → HPなど変わるものに使う
```

:::message
現場の慣習：基本的に`const`を使い、変更が必要なものだけ`let`を使う。`var`は使わない。
:::

## 3. 関数（function）

同じ処理をまとめて何度でも呼び出せる仕組みです。

```javascript

    playerHp = playerHp - damage;

        document.getElementById('message').textContent = 'ゲームオーバー！';
    } else {
        document.getElementById('message').textContent = 'HP:' + playerHp;

// 関数を呼び出す
takeDamage(10);  // HP:20
takeDamage(15);  // HP:5
```

```
関数の定義　　→　レシピを作る
関数の呼び出し→　レシピ通りに料理する（何回でもできる）
```

## 8. 動的HTML生成（createElement・appendChild）

問題が変わるたびにボタンを作り直す仕組みです。

```javascript
// ① 前のボタンを全消し
const buttonsDiv = document.getElementById('buttons');
buttonsDiv.innerHTML = '';

// ② ボタンを動的に作る
choices.forEach(function(choice) {
    const btn = document.createElement('button');       // <button>を作る
    btn.textContent = choice;                           // テキストをセット
    btn.onclick = function() { checkAnswer(choice); };  // クリック時の処理
    buttonsDiv.appendChild(btn);                        // divに追加
});
```

```
createElement → 要素を作る（まだ画面に出ない）
appendChild   → 要素を画面に追加する
innerHTML=''  → 既存の要素を全部消す（リセット）
```

## 9. 全部まとめた問題クイズ

```html
<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <title>まなびクエスト</title>
</head>
<body>
    <h1>まなびクエスト</h1>
    <p id="question"></p>
    <div id="buttons"></div>
    <p id="message"></p>

    <script>
        const questions = [
            { q: '8 × 7 = ?', a: 56, choices: [54, 56, 63, 48] },
            { q: '9 × 9 = ?', a: 81, choices: [72, 79, 81, 83] },
            { q: '144 ÷ 12 = ?', a: 12, choices: [11, 13, 12, 14] },
        ];

        let currentIndex = 0;

        function showQuestion() {
            const current = questions[currentIndex];
            document.getElementById('question').textContent = current.q;

            const buttonsDiv = document.getElementById('buttons');
            buttonsDiv.innerHTML = '';
            current.choices.forEach(function(choice) {
                const btn = document.createElement('button');
                btn.textContent = choice;
                btn.onclick = function() { checkAnswer(choice); };
                buttonsDiv.appendChild(btn);
            });
        }

        function checkAnswer(selected) {
            const correct = questions[currentIndex].a;
            if (selected === correct) {
                document.getElementById('message').textContent = '正解！';
                currentIndex++;
                if (currentIndex < questions.length) {
                    showQuestion();
                } else {
                    document.getElementById('message').textContent = '全問正解！🎉';
                }
            } else {
                document.getElementById('message').textContent = 'はずれ！';
            }
        }

        showQuestion();
    </script>
</body>
</html>
```

# Part2. Canvas API

Phaser.jsはCanvas APIの上に作られています。Phaser.jsを使う前にCanvas APIを理解しておくと、エラーが出たときに何が起きているかわかるようになります。

## Canvas APIとは？

HTMLの`<canvas>`タグ＋JavaScriptでブラウザ上に絵を描く技術です。

```html
<canvas id="myCanvas" width="600" height="400"></canvas>

<script>
    // ① キャンバスとコンテキストを取得
    const canvas = document.getElementById('myCanvas');
    const ctx = canvas.getContext('2d'); // 「2Dで描くよ」という宣言

    // ② ここから描画できる
</script>
```

### 座標系

```
(0,0) ─────────→ X（右が+）
  │
  │
  ↓
  Y（下が+）
```

左上が原点です。

---

## 10. 図形を描く

```javascript
// 四角を描く（塗りつぶし）
ctx.fillStyle = '#378ADD';
ctx.fillRect(40, 60, 160, 100); // fillRect(x, y, 幅, 高さ)

// 円を描く
ctx.fillStyle = '#1D9E75';
ctx.beginPath();                          // 新しいパスを開始
ctx.arc(300, 110, 70, 0, Math.PI * 2);   // arc(x, y, 半径, 開始角, 終了角)
ctx.fill();

// 線を描く
ctx.strokeStyle = '#D85A30';
ctx.lineWidth = 4;
ctx.beginPath();
ctx.moveTo(480, 40);  // ペンを移動（線は引かない）
ctx.lineTo(560, 180); // 線を引く
ctx.stroke();         // 画面に表示

// キャンバスを消す
ctx.clearRect(0, 0, canvas.width, canvas.height);
```

:::message
`beginPath()`は「前のパスをリセットして新規開始」する命令です。新しい形を描くときは必ず先に書きます。`fillRect()`だけは例外で不要です。
:::

### lineTo()とstroke()の違い

```
lineTo() → 「どこに線を引くか」を設定するだけ（画面には出ない）
stroke() → 設定した線を実際に画面に表示する

// 三角形の例
ctx.beginPath();
ctx.moveTo(150, 30);   // 頂点
ctx.lineTo(50, 170);   // 左下
ctx.lineTo(250, 170);  // 右下
ctx.closePath();       // 最初の点に戻る
ctx.stroke();          // 3本の線をまとめて一気に表示
```

## 11. 色・グラデーション・テキスト

```javascript
// 色の指定方法（CSSと同じ）
ctx.fillStyle = 'red';
ctx.fillStyle = '#378ADD';
ctx.fillStyle = 'rgba(55, 138, 221, 0.5)'; // 最後が透明度（0〜1）

grad.addColorStop(1,   '#D85A30'); // 1 = 終了点の色
ctx.fillStyle = grad;
ctx.fillRect(0, 0, 200, 100);

// テキスト
ctx.font = '32px sans-serif'; // サイズ + フォント
ctx.fillStyle = 'black';
ctx.fillText('まなびクエスト', 50, 100); // テキスト, x, y（左下が基準）
```

## 12. 画像を描く（スプライトシート）

```javascript
const img = new Image();
img.src = 'hero.png';

img.onload = function() {
    // 読み込み完了後に描く（onloadの外で呼ぶと何も表示されない）

    // ① そのまま描く
    ctx.drawImage(img, x, y);

    // ② サイズ指定して描く
    ctx.drawImage(img, x, y, 幅, 高さ);

    // ③ 切り抜いて描く（スプライトシート）
    ctx.drawImage(img,
        切取X, 切取Y, 切取幅, 切取高さ,  // 元画像のどこを
        描画X, 描画Y, 描画幅, 描画高さ   // どこにどのサイズで
    );
};
```

### スプライトシートとは？

キャラクターのアニメーション用コマを1枚の画像にまとめたものです。

```
┌────┬────┬────┬────┐
│歩き│歩き│攻撃│やられ│  ← 1枚の画像ファイル
│ 1  │ 2  │    │    │
└────┴────┴────┴────┘
    ↑ コマを切り替えることでアニメーションを表現
```

## 13. アニメーション（requestAnimationFrame）

```javascript
let x = 0;

function draw() {
    // ① 画面を消す
    ctx.clearRect(0, 0, canvas.width, canvas.height);

    // ② 新しい位置に描く
    ctx.fillRect(x, 70, 60, 60);

    // ③ 位置を更新
    x += 2;
    if (x > canvas.width) x = -60;

    // ④ 次のフレームを予約（ループ）
    requestAnimationFrame(draw);
}
draw(); // 最初の1回を呼ぶ
```

### requestAnimationFrameとsetTimeoutの違い

| | setTimeout | requestAnimationFrame |
|---|---|---|
| タイミング | 指定ms後 | ブラウザの描画に同期 |
| 滑らかさ | ズレが出る | 滑らか（毎秒60回） |
| タブ非表示時 | 動き続ける | 自動停止（省エネ） |

ゲームには必ず`requestAnimationFrame`を使います。

### フレームカウンタでアニメーションを制御する

```javascript
let frame = 0;

function draw() {
    // 8フレームごとにコマを切り替える
    const コマ = Math.floor(frame / 8) % コマ数;

    frame++;
    requestAnimationFrame(draw);
}
```

## 14. キー入力でキャラを動かす

```javascript
// フラグ方式（滑らかに動く）
const keys = {};
document.addEventListener('keydown', e => { keys[e.key] = true; });
document.addEventListener('keyup',   e => { keys[e.key] = false; });

let x = 300, y = 150;
const speed = 3;

function draw() {
    ctx.clearRect(0, 0, canvas.width, canvas.height);

    // 毎フレーム、キーの状態を見て座標を更新
    if (keys['ArrowRight']) x += speed;
    if (keys['ArrowLeft'])  x -= speed;
    if (keys['ArrowUp'])    y -= speed;
    if (keys['ArrowDown'])  y += speed;

    // 壁の当たり判定（画面外に出ないように）
    x = Math.max(20, Math.min(canvas.width  - 20, x));
    y = Math.max(20, Math.min(canvas.height - 20, y));

    ctx.fillRect(x, y, 40, 40);
    requestAnimationFrame(draw);
}
draw();
```

:::message
キーを押した瞬間だけ動かすのではなく、フラグ方式（押している間ずっとtrueにする）にすることで滑らかな動きになります。
:::

# Part3. Phaser.js

## Phaser.jsとは？

ブラウザで動く2Dゲームを作るための**ゲーム専用JavaScriptライブラリ**です（2013年〜、MIT License）。

### なぜCanvas APIだけではなくPhaser.jsが必要？

| 機能 | Canvas APIだけ | Phaser.js |
|------|---------------|-----------|
| キャラを動かす | 自分で書く | 組み込み済み |
| 衝突判定 | 自分で計算 | 1行で書ける |
| 重力・ジャンプ | 自分で計算 | 組み込み済み |
| アニメーション | 自分で管理 | 組み込み済み |
| シーン切り替え | 自分で管理 | 組み込み済み |

Canvas APIで学んだことはPhaser.jsの土台になっています。

```
あなたのコード
    ↓
Phaser.js（Canvas APIをラップしている）
    ↓
Canvas API
    ↓
ブラウザ画面
```

## 15. Phaser.jsの基本構造

Phaser.jsは3つの概念で構成されています。

```javascript
// ① Sceneを作る（ゲームの1画面）
class GameScene extends Phaser.Scene {

    constructor() {
        super('GameScene'); // シーンの名前
    }

    preload() {
        // 画像・音声の読み込み（ゲーム開始前に1回だけ実行）
        this.load.image('hero', 'assets/hero.png');
    }

    create() {
        // 最初に1回だけ実行（キャラ配置など）
        this.hero = this.add.sprite(300, 200, 'hero');
    }

    update() {
        // 毎フレーム実行（キー入力・移動など）
        // Canvas APIの requestAnimationFrame と同じ役割
    }
}

// ② Configを作る（ゲームの設定）
const config = {
    type: Phaser.AUTO,         // Canvas or WebGL を自動選択
    width: 640,
    height: 480,
    backgroundColor: '#1a1a2e',
    physics: {
        default: 'arcade',
        arcade: {
            gravity: { y: 0 }, // トップビューRPGは重力なし
            debug: false,
        }
    },
    scene: [TitleScene, FieldScene, BattleScene], // 複数シーン
};

// ③ Gameを起動する
const game = new Phaser.Game(config);
```

### Sceneの3メソッド実行順

```
ゲーム起動
   ↓
preload()  → 画像・音声を読み込む
   ↓
create()   → キャラ・マップを配置する（1回だけ）
   ↓
update()   → 毎フレーム実行（ループ）
update()
update()
  ...
```

### Canvas APIとの対応

```
Canvas API               Phaser.js
──────────────────────────────────
new Image(); img.src = → preload()
最初の描画             → create()
requestAnimationFrame  → update()
ctx.drawImage()        → this.add.sprite()
```

## 16. スプライト表示・移動

```javascript
create() {
    // スプライトを配置
    // add.sprite(x, y, キー)
    this.hero = this.add.sprite(300, 150, 'hero');
}

update() {
    // 毎フレーム右に移動
    this.hero.x += 2;

    // 画面外に出たら左に戻す
    if (this.hero.x > 580) this.hero.x = -20;

    // 回転させる（Canvas APIより簡単）
    this.hero.angle += 2;
}
```

### Canvas APIとの違い

| やること | Canvas API | Phaser.js |
|----------|-----------|-----------|
| 画面を消す | `ctx.clearRect()` | 自動（不要） |
| ループ | `requestAnimationFrame()` | `update()`が代わり |
| 回転 | 自分で計算 | `sprite.angle += 2` |

## 17. キー入力（Phaser.js版）

```javascript
create() {
    // ① カーソルキーを登録（これだけでOK！）
    this.cursors = this.input.keyboard.createCursorKeys();
    this.hero = this.physics.add.sprite(300, 150, 'hero');
    this.speed = 160;
}

update() {
    // ② 毎フレーム、キーの状態を確認して速度をセット
    const vx = this.cursors.right.isDown ? this.speed
             : this.cursors.left.isDown  ? -this.speed : 0;
    const vy = this.cursors.down.isDown  ? this.speed
             : this.cursors.up.isDown    ? -this.speed : 0;

    this.hero.setVelocity(vx, vy);

    // 左右反転
    if (vx !== 0) this.hero.scaleX = vx > 0 ? 1 : -1;

    // 画面外制限
    // Phaser.Math.Clamp = Math.max + Math.min と同じ
    this.hero.x = Phaser.Math.Clamp(this.hero.x, 20, 540);
    this.hero.y = Phaser.Math.Clamp(this.hero.y, 20, 270);
}
```

### Canvas APIとの違い

| Canvas API | Phaser.js |
|------------|-----------|
| `addEventListener` 2行 | `createCursorKeys()` 1行 |
| `Math.max` + `Math.min` | `Phaser.Math.Clamp()` 1行 |

## 18. 衝突判定

```javascript
create() {
    // 物理エンジン付きスプライト（add → physics.add に変えるだけ）
    this.hero  = this.physics.add.sprite(80,  150, 'hero');
    this.enemy = this.physics.add.sprite(400, 150, 'enemy');

    // 敵を自動で動かす
    this.enemy.setVelocity(120, 80);        // 速度を設定
    this.enemy.setBounce(1, 1);             // 壁で跳ね返る（1 = 完全反射）
    this.enemy.setCollideWorldBounds(true); // 画面外に出ない

    // 衝突判定を登録
    // overlap：重なったら onHit を呼ぶ（すり抜ける）
    this.physics.add.overlap(
        this.hero, this.enemy, this.onHit, null, this
    );
}

onHit(hero, enemy) {
    // 敵に当たったとき → バトル開始！
    this.scene.start('BattleScene', { enemy: 'スライム' });
}
```

### overlapとcolliderの違い

| メソッド | 動作 | 用途 |
|----------|------|------|
| `physics.add.overlap()` | 重なってもすり抜ける | 敵との接触判定 |
| `physics.add.collider()` | ぶつかって止まる | 壁・障害物 |

## 19. シーン切り替え

まなびクエストのタイトル→フィールド→バトルの流れです。

```javascript
// データを渡しながらシーンを切り替える
this.scene.start('BattleScene', { enemy: 'スライム', hp: 3 });

// 渡されたデータを受け取る
create(data) {
    console.log(data.enemy); // 'スライム'
    console.log(data.hp);    // 3
}
```

### まなびクエストのシーン構成

```
TitleScene  → タイトル画面
    ↓ スタートボタン
FieldScene  → フィールド（歩き回る）
    ↓ ランダムエンカウント（歩くたびに一定確率で発生）
BattleScene → バトル（問題が出る）
    ↓ 正解 → 敵にダメージ / 不正解 → 反撃
FieldScene  → フィールドに戻る
```

### ランダムエンカウントの実装イメージ

ドラクエ1のように草むらを歩いていると一定確率で敵が出る仕組みです。

```javascript
update() {
    const moving = vx !== 0 || vy !== 0;

    if (moving) {
        // 毎フレーム0.5%の確率でエンカウント
        if (Phaser.Math.Between(1, 200) === 1) {
            this.scene.start('BattleScene', { enemy: 'スライム' });
        }
    }
}
```
