## CI/CDとは？

### CI（継続的インテグレーション）

```
コードをpushする
　　↓
自動でテストが走る
　　↓
テスト合格 ✅ or 失敗 ❌ を通知
```

### CD（継続的デリバリー）

```
テストが合格したら
　　↓
自動で本番環境にデプロイ
```

### なぜCI/CDが重要か

```
【CI/CDなし】
コードを変更 → 手動でテスト → 手動でデプロイ
→ 時間がかかる・ミスが起きやすい😭

【CI/CDあり】
コードをpush → 自動でテスト → 自動でデプロイ
→ 速い・ミスが減る ✅
```

## GitHub Actionsとは？

GitHubに組み込まれているCI/CDツールです。

```
【メリット】
・GitHubと完全統合
・無料枠あり（月2000分）
・設定ファイル1つで動く
・現場で最もよく使われている
```

### 今回やること

```
GitHubにpushする
　　↓
GitHub Actionsが自動でpytestを実行
　　↓
全テスト合格 → ✅ グリーン
テスト失敗   → ❌ レッド
```

## 実装手順

### 1. ワークフローファイルを作成

```bash
mkdir -p .github/workflows
```

#### .githubフォルダの意味

`.`（ドット）で始まるフォルダは**隠しフォルダ**です。

```
通常のフォルダ → todos/、config/　← lsコマンドで表示される
隠しフォルダ　 → .github/、.git/　← lsコマンドでは表示されない
```

GitHubは`.github/workflows/`の中の`.yml`ファイルを自動で読み込んでActionsを実行します。

### 2. django.ymlを作成

```yaml
name: Django CI

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest

    services:
      postgres:
        image: postgres:16
        env:
          POSTGRES_DB: django_todo
          POSTGRES_USER: django_user
          POSTGRES_PASSWORD: django_password
        ports:
          - 5432:5432
        options: >-
          --health-cmd pg_isready
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5

    steps:
      - uses: actions/checkout@v4

      - name: Pythonのセットアップ
        uses: actions/setup-python@v5
        with:
          python-version: '3.14'

      - name: ライブラリのインストール
        run: pip install -r requirements.txt

      - name: テストの実行
        env:
          SECRET_KEY: 'django-insecure-test-secret-key'
          DEBUG: 'True'
          POSTGRES_DB: django_todo
          POSTGRES_USER: django_user
          POSTGRES_PASSWORD: django_password
          DATABASE_HOST: localhost
        run: pytest
```

#### 各設定の意味

| 設定 | 意味 |
|---|---|
| `name` | ワークフローの名前 |
| `on: push` | mainブランチにpushしたとき実行 |
| `on: pull_request` | mainへのPRを作ったとき実行 |
| `runs-on: ubuntu-latest` | GitHubが用意するUbuntuで実行 |
| `services: postgres` | テスト用のPostgreSQLを起動 |
| `steps` | 実行するコマンドを順番に定義 |

### 3. settings.pyのDB設定を修正

GitHub ActionsではPostgreSQLのホスト名が`localhost`になります。環境変数で切り替えられるようにします。

```python
# config/settings.py
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql',
        'NAME': os.environ.get('POSTGRES_DB'),
        'USER': os.environ.get('POSTGRES_USER'),
        'PASSWORD': os.environ.get('POSTGRES_PASSWORD'),
        'HOST': os.environ.get('DATABASE_HOST', 'db'),  # ← 修正
        'PORT': '5432',
    }
}
```

```
Docker環境　　　　→　DATABASE_HOST未設定 → デフォルト値'db'を使用 ✅
GitHub Actions　→　DATABASE_HOST=localhost → localhostを使用 ✅
```

## トラブルシューティング

### could not translate host name "db" to address

**原因**

GitHub Actionsの環境ではDockerのサービス名`db`が使えない。

**解決方法**

```yaml
# django.ymlのテスト実行部分に追加
env:
  DATABASE_HOST: localhost  # ← 追加
```

```python
# settings.pyのHOSTを環境変数から読み込む
'HOST': os.environ.get('DATABASE_HOST', 'db'),
```
