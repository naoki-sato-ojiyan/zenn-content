## REST APIとは？

### そもそもAPIとは？

**Application Programming Interface**の略で、「アプリ同士が会話するための窓口」です。

```
【人間がWebを使う場合】
人間 → ブラウザ → サーバー → HTML → ブラウザが表示

【アプリがAPIを使う場合】
アプリ → API → サーバー → JSON → アプリが処理
```

### JSONとは？

APIが返すデータの形式です。HTMLと違ってデザインがなく、**データだけ**が入っています。

```json
{
  "id": 1,
  "title": "買い物",
  "completed": false,
  "created_at": "2026-06-08"
}
```

### RESTとは？

APIの設計ルールのことです。URLとHTTPメソッドでCRUDを表現します。

| 操作 | HTTPメソッド | URL | 意味 |
|---|---|---|---|
| 一覧取得 | GET | /api/todos/ | Todo全件取得 |
| 1件取得 | GET | /api/todos/1/ | ID=1のTodo取得 |
| 作成 | POST | /api/todos/ | Todo新規作成 |
| 更新 | PUT | /api/todos/1/ | ID=1のTodo更新 |
| 削除 | DELETE | /api/todos/1/ | ID=1のTodo削除 |

### 今のToDoアプリとの違い

```
【今のToDoアプリ】
ブラウザ → /create/ → Django → DBに保存 → HTMLを返す → ブラウザに表示

【REST API化後】
スマホアプリ ┐
ブラウザ     ├→ POST /api/todos/ → Django → DBに保存 → JSONを返す
他サービス  ┘
```

**1つのAPIで複数のクライアントに対応できる**のが最大のメリットです！

## Django REST Frameworkとは？

DjangoでREST APIを簡単に作れるライブラリです。

```
Django単体　　→　HTMLを返すWebアプリ
Django + DRF　→　JSONを返すREST API ✅
```

### DRFの3つの主要コンポーネント

#### ① Serializer（シリアライザー）

モデルのデータをJSONに変換する役割です。

```python
# Pythonオブジェクト
todo = Todo(id=1, title="買い物", completed=False)

# Serializerが変換する ↓

# JSON
{
  "id": 1,
  "title": "買い物",
  "completed": false
}
```

逆にJSONをPythonオブジェクトに変換することもできます（バリデーションも自動）。

#### ② ViewSet（ビューセット）

CRUD操作をまとめて定義できる仕組みです。

```python
【通常のviews.py】
def todo_list(request):     # 一覧
def todo_create(request):   # 作成
def todo_delete(request):   # 削除
def todo_complete(request): # 更新
# 4つの関数を個別に書く必要がある

【DRFのViewSet】
class TodoViewSet(viewsets.ModelViewSet):
    queryset = Todo.objects.all()
    serializer_class = TodoSerializer
# これだけで一覧・作成・更新・削除が全部できる！
```

#### ③ Router（ルーター）

URLを自動生成する仕組みです。

```python
router = DefaultRouter()
router.register('todos', TodoViewSet)

# 以下のURLが自動で生成される
# GET    /api/todos/       → 一覧
# POST   /api/todos/       → 作成
# GET    /api/todos/1/     → 1件取得
# PUT    /api/todos/1/     → 更新
# DELETE /api/todos/1/     → 削除
```

## 実装手順

### 1. DRFをインストール

```bash
docker compose exec web pip install djangorestframework
pip freeze > requirements.txt
```

### 2. settings.pyに登録

```python
INSTALLED_APPS = [
    ...
    'rest_framework',  # 追加
    'todos.apps.TodosConfig',
]
```

### 3. Serializerを作成

```python
# todos/serializers.py
from rest_framework import serializers
from .models import Todo

class TodoSerializer(serializers.ModelSerializer):
    class Meta:
        model = Todo
        fields = ['id', 'title', 'completed', 'created_at']
        read_only_fields = ['created_at']
```

#### 各設定の意味

| 設定 | 意味 |
|---|---|
| `model = Todo` | どのモデルを使うか |
| `fields = [...]` | JSONに含めるフィールドを指定 |
| `read_only_fields` | 読み取り専用のフィールド（外部から変更不可） |

:::message
`user`フィールドは含めていません。ログイン済みのユーザーが自分のTodoを取得する場合、「自分のTodoなのは当然」なのでuser情報は不要です。不要な情報を返さないのがAPIの基本ルールです。
:::

### 4. ViewSetを作成

```python
# todos/api_views.py
from rest_framework import viewsets, permissions
from .models import Todo
from .serializers import TodoSerializer

class TodoViewSet(viewsets.ModelViewSet):
    serializer_class = TodoSerializer
    permission_classes = [permissions.IsAuthenticated]

    # ログインユーザーのTodoのみ取得
    def get_queryset(self):
        return Todo.objects.filter(user=self.request.user)

    # 作成時にログインユーザーを自動セット
    def perform_create(self, serializer):
        serializer.save(user=self.request.user)
```

#### 各設定の意味

| 設定 | 意味 |
|---|---|
| `viewsets.ModelViewSet` | CRUD操作を全部自動で使えるようになる |
| `permission_classes = [IsAuthenticated]` | 未ログインのユーザーはAPIにアクセスできない（@login_requiredのAPI版） |
| `get_queryset()` | ログインユーザーのTodoだけを返す |
| `perform_create()` | Todo作成時にログインユーザーを自動でセット |

### 5. URLを設定

```python
# config/urls.py
from django.contrib import admin
from django.urls import path, include
from rest_framework.routers import DefaultRouter
from todos.api_views import TodoViewSet

router = DefaultRouter()
router.register('todos', TodoViewSet, basename='todo')

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', include('todos.urls')),
    path('api/', include(router.urls)),  # APIのURL
    path('accounts/', include('django.contrib.auth.urls')),
]
```

## トラブルシューティング

### ImportError: cannot import name 'DefaultRouter' from 'rest_framework'

**原因**

```python
# ❌ 間違い
from rest_framework import DefaultRouter

# ✅ 正しい
from rest_framework.routers import DefaultRouter
```

`DefaultRouter`は`rest_framework`直下ではなく`rest_framework.routers`の中にあります。
