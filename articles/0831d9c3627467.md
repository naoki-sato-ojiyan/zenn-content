## テストコードとは？

**コードが正しく動くかを自動で確認するプログラム**です。

```
【テストなし】
コードを変更する
　　↓
手動でブラウザで確認する
　　↓
「あ、別の機能が壊れた😭」← よくある

【テストあり】
コードを変更する
　　↓
pytest を実行する
　　↓
自動で全機能をチェック ✅
「全部OK！」or「ここが壊れた！」と即座にわかる
```

### 現場でテストが重要な理由

```
① 品質保証　　→　バグを早期発見できる
② リファクタ　→　安心してコードを改善できる
③ チーム開発　→　他の人のコードが壊れていないか確認できる
```

---

## pytestとは？

Pythonのテストフレームワークです。Djangoの標準テストより書きやすく、現場でよく使われています。

| | Django標準テスト | pytest |
|---|---|---|
| 記法 | クラスベース | 関数ベース（シンプル） |
| フィクスチャ | setUp/tearDown | @pytest.fixture |
| 現場での使用 | △ | ◎ |

## 実装手順

### 1. インストール

```bash
docker compose exec web pip install pytest pytest-django
docker compose exec web pip freeze > requirements.txt
```

### 2. pytest.iniを作成

プロジェクトのルートフォルダに`pytest.ini`を作成します。

```ini
[pytest]
DJANGO_SETTINGS_MODULE = config.settings
python_files = tests.py test_*.py *_test.py
```

#### pytest.iniが必要な理由

```
pytestはDjangoのことを知らない
→ 「どのsettings.pyを使えばいいか」を教える必要がある
→ これがないとDjangoの設定が読み込めずエラーになる
```

### 3. テストコードを書く

```python
# todos/tests.py
import pytest
from django.contrib.auth.models import User
from .models import Todo


# テスト用のユーザーを作成するフィクスチャ
@pytest.fixture
def user(db):
    return User.objects.create_user(
        username='testuser',
        password='testpass123'
    )


# テスト用のTodoを作成するフィクスチャ
@pytest.fixture
def todo(db, user):
    return Todo.objects.create(
        user=user,
        title='テストTodo',
        completed=False
    )


# Todoが正しく作成されるかテスト
def test_todo_create(todo):
    assert todo.title == 'テストTodo'
    assert todo.completed == False


# Todoの完了切り替えをテスト
def test_todo_complete(todo):
    todo.completed = not todo.completed
    todo.save()
    assert todo.completed == True


# Todoの削除をテスト
def test_todo_delete(todo):
    todo_id = todo.id
    todo.delete()
    assert Todo.objects.filter(id=todo_id).count() == 0


# ユーザーごとのTodo管理をテスト
def test_todo_user_isolation(db, user):
    other_user = User.objects.create_user(
        username='otheruser',
        password='testpass123'
    )
    Todo.objects.create(user=user, title='自分のTodo')
    Todo.objects.create(user=other_user, title='他人のTodo')

    my_todos = Todo.objects.filter(user=user)
    assert my_todos.count() == 1
    assert my_todos.first().title == '自分のTodo'
```

## コードの意味

### フィクスチャとは？

**テストの前に準備するデータや設定を定義する仕組み**です。

```python
@pytest.fixture
def user(db):
    return User.objects.create_user(...)
```

```
@pytest.fixture → 「これはフィクスチャですよ」という印
db             → DBを使うテストに必要な引数（pytest-djangoが提供）
```

テスト関数の引数にフィクスチャ名を書くだけで自動的に呼び出されます。

```python
def test_todo_create(todo):  # todoフィクスチャが自動で呼ばれる
    assert todo.title == 'テストTodo'
```

### assertとは？

**条件が正しいかチェックする命令**です。

```python
assert todo.title == 'テストTodo'
# todo.titleが'テストTodo'と等しければ → テスト合格 ✅
# todo.titleが'テストTodo'と等しくなければ → テスト失敗 ❌
```

### テスト結果の見方

```
.　→　テスト合格 ✅
F　→　テスト失敗 ❌
E　→　エラー 🔴
```

## テストを実行する

```bash
docker compose exec web pytest
```

### 実行結果

```
collected 4 items
todos/tests.py ....   [100%]

4 passed in 1.23s
```

`....`の4つのドットが4つのテスト全て合格を意味しています！

### 合格したテスト

```
✅ test_todo_create        → Todoが正しく作成される
✅ test_todo_complete      → 完了切り替えが動く
✅ test_todo_delete        → 削除が動く
✅ test_todo_user_isolation → ユーザーごとのTodo管理が正しい
```
